
# Day-04-Default-Gateway-Configuration

## Objective

Understand the purpose of a default gateway and troubleshoot gateway-related connectivity issues.

## Topics Covered

- Default Gateway
- Router Forwarding
- End-to-End Connectivity
- Basic Troubleshooting

## Problem

PC1 could not communicate with PC2 because an incorrect default gateway was configured.

## Root Cause

Gateway was configured as 192.168.1.100 instead of 192.168.1.1.

## Resolution

Configured the correct gateway.

## Outcome

Successfully restored communication between subnets.
